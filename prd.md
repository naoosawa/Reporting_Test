# PRD: Reporting機能（1st プロジェクト）

**機能名**: 【Reporting】Reporting機能  
**バージョン**: Ver1.0  
**ステータス**: 開発中  
**開発開始予定日**: 2026-02-16  
**リリース目標**: 2026-04-30（6月有報開示に使用）

---

## 前提条件と目的整理（Why / Scope）

### 概要

1stプロジェクトでは、**過去の開示資料を取り込み、開示物の構成に沿って「表」と「テキスト編集」の箱（編集可能なドラフト）を自動生成できる状態**を実現する。

あわせて、Consolidation等で集計された定量指標データを**表に反映**できるようにし、開示物の更新・転記の負荷とミスを大幅に削減する。

### 全体スケジュール

- 開発期間: 2/16〜4/30（54営業日）
- 実装のみで60人日の想定（バッファなし）
- 2名体制の場合でも、要件把握・設計・QA・仕様調整等を含めるとタイトスケジュール

---

## Goals（この取組みのゴール）

**《1st プロジェクト》過去開示取り込みを起点に、表の最新化と最小限のテキスト編集、外部出力ができる状態**

- **Goal**: 過去開示物を基に、表作成箇所とテキスト編集箇所を含むレポート生成機能を用意し、定量指標の最新反映と編集を行って出力できる状態
- **リリース目標**: 2026年4月末（6月有報開示に使う）

### 本取り組みで解決を目指すこと

- 過去開示物の構成に沿ったドラフトを起点に、更新作業を「差し替え」中心へ寄せる（ゼロから作らない）
- 表の数値更新・転記に伴う工数とミスを削減する
- 開示物ごとの重複した数値管理を解消する
- 更新が入った箇所を把握し、再確認・再出力の判断ができる状態を作る
- **過去開示資料のインポート**
  - 過去の開示資料をインポートし、編集可能な形式に変換
  - 経年更新箇所の自動修正

### 対象範囲（Scope）

**対象データ**
- 過去開示資料（アップロードした原本）
- Consolidationで集計されたE/S/Gの定量指標データ（表の反映対象）

**アウトプット**
- 過去開示資料から生成されたレポート（表枠＋テキスト編集箇所を含む）
- 定量指標データをもとに生成・更新される表
- Word, Excel, PDF形式でのエクスポート

**利用シーン**
- 開示物の更新・経年修正（過去資料の取り込み→更新）
- 社外向け開示資料の図表（表）準備と原稿の最小編集
- 社内向け報告資料作成時の定量指標の整理（表中心）

## Background（取り組みの背景）

### なぜReportingに取り組むのか

- SmartESG上には、Consolidation・Database・Focus・Benchmarkを通じて、サステナビリティに関する定量・定性データが一元的に蓄積されつつある
- 一方で、これらの情報は**社内外の報告物作成という最終アウトプットには直接つながっていない**
- その結果、SmartESGの価値が「分析・可視化」で止まり、報告資料作成という最終成果につながらない
- このため、顧客は別ツールでの再加工が必要となり、**SmartESGへの依存度が高まらない**

### ユーザー（企業側）が直面している現状課題

- **集計された定量指標を、報告用途にそのまま使えない**
  - 有価証券報告書や統合報告書向けに、毎回グラフ・表を作り直している
  - 指標定義や表現が属人化し、再利用しづらい
- **最新データ反映に工数とリスクが伴っている**
  - 数値の転記・更新が手作業中心である
  - 更新漏れや転記ミスが発生しやすい
- **過去開示物の更新・経年修正が重い**
  - 過去資料を参照しながらの修正作業に時間がかかる
  - 修正箇所の抜け漏れが起きやすい

### なぜ1stプロジェクトでは定量指標に集中するのか

Consolidationユーザーには明確な期限を持つ開示業務があり、定量指標は確実に利用されるニーズがある。また、社内的にも実現可能性が高いため、1stプロジェクトではConsolidationの指標データを用いた表の生成、レポート出力に集中する。

### 本取り組みの位置づけ

- Reportingは、SmartESGのデータ・分析を**社内外の報告物という最終成果物につなげるための出口**として位置づけられる
- 1stプロジェクトは、その中でも**「定量指標を、使える形で出せる状態を作る」最初のフェーズ**である
- 本フェーズで基盤を整えることで、Reporting機能全体を段階的に拡張していくことを可能にする

## 解決策の整理（What）

### User Stories（ユーザーストーリー）

- **過去開示資料を起点に、レポートのドラフトを素早く作りたい**
  - サステナビリティ担当者として、集計された定量指標を、報告資料の構成に沿ったドラフトとして用意したい。なぜなら、報告資料作成をゼロから組み立てる作業をなくしたいからだ。

- **KPIの見せ方を出力前に確定したい**
  - サステナビリティ担当者として、KPIの表示期間や単位、桁数の見せ方をSmartESG Reporting上で決めたい。なぜなら、報告資料作成時の判断をSmartESG内で完結させたいからだ。

- **表とテキストを最小限の編集で整えたい**
  - サステナビリティ担当者として、自動生成された表枠とテキスト編集箇所を、必要な範囲だけ編集してドラフトを完成させたい。なぜなら、制作会社に渡す原稿を短時間で作りたいからだ。

- **確定したドラフトをそのまま出力したい**
  - サステナビリティ担当者として、SmartESG上で整理した内容を、レポート（ドラフト）を、Word・Excel・PDFでそのまま出力したい。なぜなら、出力後は装飾などのスライド作成に注力したいからだ。

- **最新の情報更新を確認してレポートを出力したい**
  - サステナビリティ担当者として、Consolidation等の定量指標に更新が入った箇所を把握したうえで再出力したい。なぜなら、更新漏れや転記ミスを防ぎたいからだ。

### UseCase（ユースケース）

| ID | 内容 |
|----|------|
| UC-01 | 過去開示資料を起点に、レポート（ドラフト）を準備する |
| UC-02 | KPIの表示単位・桁数・期間を指定して表示方法を確定する |
| UC-03 | 報告用途に応じて、図表（表）を配置・構成する |
| UC-04 | ドラフトのテキスト編集を行い、最小限の手直しで完成させる（Ver2） |
| UC-05 | 確定したドラフトをWord・Excel・PDF形式で出力する |
| UC-06 | 定量指標の最新更新を確認し、最新版を反映してレポートを出力する |
